# 正边准确率 (pos_acc) 快速下降分析

## 现象

单图 ACM 训练时，`pos_acc`（存在边上的预测准确率）在少数 epoch 内从 ~0.6 掉到 0，而 `neg_acc`（非边上的准确率）很快接近 1.0：

- Epoch 0: pos_acc 0.62, neg_acc 0.99  
- Epoch 2: pos_acc 0.92, neg_acc 0.99  
- Epoch 4: pos_acc 0.07, neg_acc 0.999  
- Epoch 5+: pos_acc 0.00, neg_acc 1.00  

即模型很快变成「几乎全预测为无边」。

---

## 原因

1. **正负极不平衡**  
   query 边里「有边」的占比很小（例如 2%–10%），「无边」占绝大多数。BCE 对每条边独立，梯度被大量负样本主导。

2. **局部最优**  
   若模型对所有 query 都预测「无边」：  
   - 负样本几乎全对 → neg_acc ≈ 1，exist_BCE 里负项很小；  
   - 正样本全错 → pos_acc = 0，但正样本数量少，对总 BCE 的贡献被负样本盖过。  
   因此从 BCE 角度，「全预测负」是一个很容易掉进去的局部最优。

3. **没有对正类加权**  
   当前存在性头用的是普通 BCE，没有对「有边」类做 `pos_weight` 或类似加权，正类梯度相对更弱，模型更容易往「全负」塌。

---

## 可做的缓解

1. **对存在性 BCE 做正类加权**  
   例如设 `pos_weight = (num_neg / num_pos)` 或略小，让正样本的梯度权重大一些，减轻「全预测负」的诱惑。

2. **提高 query 里正边比例**  
   若 `edge_fraction` 或 query 采样里正边比例能适当提高（在显存允许下），正样本数变多，BCE 更平衡，pos_acc 不易直接塌到 0。

3. **用 Focal Loss 或带 margin 的损失**  
   对难负样本降权或对正类加 margin，让模型更关注「该有边却预测成无边」的误差。

4. **监控并早停**  
   若发现 pos_acc 持续掉到接近 0 而 neg_acc 接近 1，可以提前停或回退到更早的 checkpoint，并同时加上面 1/2/3 之一再训。

---

## 小结

pos_acc 快速下降主要是 **query 上正负极不平衡 + 普通 BCE** 导致模型很快收敛到「全预测无边」的局部最优。优先建议在存在性头上加 **pos_weight**（或提高 query 正边比例），再视情况考虑 Focal/margin 等。
