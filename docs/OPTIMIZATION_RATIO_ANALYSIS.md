# 优化比波动原因与改进

## 1. 现象

- `train_loss/优化程度` = L(pred, ideal) / (L(noisy, ideal) + ε) 波动很大，无明显收敛趋势。
- 理论上应 < 1（预测比噪声更接近理想图），但经常出现 > 1 甚至 > 10 的尖峰。
- 即使 < 1 时，生成效果也不理想。

## 2. 原因分析

### 2.1 分母 L(noisy, ideal) 随 t 变化剧烈

- **t 小**（噪声弱）：noisy 接近 clean，也接近 ideal → L(noisy, ideal) 很小 → ratio = L_pred / (很小) 会**爆炸**（尖峰）。
- **t 大**（噪声强）：noisy 离 ideal 远 → L(noisy, ideal) 大 → ratio 容易 < 1。
- 每步随机采一个 t，所以 step 之间分母差异很大，比值自然剧烈波动。

### 2.2 目标在变（ideal 每步不同）

- ideal_at_t 由**当前步**的 noisy 和 t 算出来，每步的「理想图」都不同。
- 模型在追一个**移动目标**，难以形成稳定下降趋势。

### 2.3 单步单样本

- batch_size=1，每步只有一个 t、一次 query 采样，没有对多个 t 或多次 query 平均。
- 梯度噪声大，曲线会非常抖。

### 2.4 即使 ratio < 1 效果也不好的可能原因

- **只改 query 边**：每步只更新一小块边，全图结构要很多步才慢慢变好；单步 ratio 小不代表整图已经接近 ideal。
- **ideal 是近似**：ideal 是梯度几步算出来的近似，不是真正最优；拟合这个近似不一定等于生成质量好。
- **训练与采样不一致**：训练是「随机 t + 随机 query」的单步去噪，采样是整条轨迹 + 多轮 merge；单步指标和最终生成质量不是一一对应。

## 3. 为什么会出现极端大值（ratio >> 1）

你的直觉是：**分母小 = 可优化空间小 = 模型施展空间有限，那分子（L(pred, ideal)）为什么能偏差到那么大？**

要点在于「分母小」和「分子大」各自代表什么、以及它们是否矛盾。

### 3.1 分母小代表什么

- **L(noisy, ideal) 小**：当前步的 **noisy 在「全图结构」上已经和 ideal 很接近**。
- 也就是说：**noisy 在「这一步、这一组 query」上的赋值，恰好让 relation matrix / metapath2 / metapath3 / degree 等全局统计和 ideal 很接近**。
- 所以「可优化空间小」的准确含义是：**从 noisy 到 ideal 能带来的结构改进幅度有限**（天花板低），而不是「模型只能做很小的改动」。

### 3.2 分子大代表什么

- **L(pred, ideal) 大**：把 **query 上的边换成模型预测** 后，得到的全图结构和 ideal **差很多**。
- pred 与 noisy 的**唯一**区别就是 **query 上的边/类型**；非 query 部分三者（noisy / pred / ideal）都一样。
- 所以分子大 = **模型在「同一批 query 位置」上给出的预测，在全局结构意义下离 ideal 很远**，甚至比「保留 noisy」还差。

### 3.3 为何不矛盾：分母小 ≠ 模型被限制住

- **分母小**：说明 **noisy 在这一步「运气好」或 query 影响有限**，所以 L(noisy, ideal) 小。
- **模型并没有被限制**：它仍然要对 query 上的每条边做预测；它完全可以预测成和 ideal 完全不同的边/类型。
- 一旦预测错了，而且错的方式**对全局结构很敏感**（例如错类型导致 relation matrix 偏很多、或 metapath 计数差很多），**L(pred, ideal) 就会很大**。
- 因此：**「可优化空间小」只说明「noisy 已经不错」；模型仍可能在这一步做出「比 noisy 差很多」的预测**，从而分子大、ratio 爆炸。

### 3.4 为何 pred 能比 noisy 差这么多（同一批 query）

- **noisy**：来自扩散过程，在这一步是「随机/噪声」在 query 上的赋值，**有可能恰好**在类型分布、关键 metapath 上接近 ideal（所以 L_noisy 小）。
- **pred**：是模型**主动预测**的；若模型还没学好，或这一步 t/query 较难，可能**系统性地**预测错类型、错边，从而在 relation matrix / metapath / degree 上产生**大偏差**。
- **结构度量的敏感度**：同一批 query 边，**错在「关键类型」或「关键位置」**时，对 relation matrix、metapath 等的影响可以远大于「随机错」；noisy 像是随机错，pred 可能是系统错，所以 L_pred 可以远大于 L_noisy。

### 3.5 小结（不改代码下的结论）

- **分母小**：这一步 noisy 已经接近 ideal，可优化空间小，**只说明「天花板低」**。
- **分子大**：模型在这一步的预测在结构上离 ideal 很远，**说明「模型预测错了，而且错得对全局结构影响很大」**。
- 两者不矛盾：**模型并没有因为「可优化空间小」而被限制输出**；它仍然可以输出很差的 pred，从而在「同一批 query、同一套结构度量」下得到很大的 L(pred, ideal)，导致 ratio 出现极端大值。
- 若要缓解尖峰，可从「分母下限、对 t 加权、梯度裁剪」等方向做（见下文），此处仅做原因分析。

## 4. 已做改进

### 4.1 分母下限（防尖峰）

- 配置项：`structure_ratio_denominator_floor`（默认 0.01）。
- 计算：`ratio = L_pred / max(L_noisy, floor)`，避免 L_noisy 极小时比值爆炸。
- 可在 `configs/model/discrete.yaml` 或实验 yaml 中覆盖。

### 4.2 可选：记录 L_pred、L_noisy 便于排查

- 若在 `train_loss.forward` 里对 use_ratio 分支打 log：`train_loss/L_pred`、`train_loss/L_noisy`，可在 wandb 中看分母何时过小导致尖峰。

## 5. 可进一步尝试

- **学习率 / 梯度裁剪**：略降 LR 或加强 grad_clip，减轻单步更新过大带来的震荡。
- **对 t 加权**：例如只在小 t 用较小权重或跳过，减少「分母很小」的 step 对梯度的主导。
- **多步平均**：同一 step 内对多个 t 或多次 query 取平均再算 ratio（计算量会上去）。
- **评估方式**：更多以**最终生成图**的结构指标（或下游任务）为准，而不是只看训练曲线是否平滑。
