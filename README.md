# DiHuG

DiHuG is a heterogeneous graph diffusion project built on top of SparseDiff-style discrete diffusion. It focuses on graph generation for typed nodes and typed relation families, with local block queries and a pyHGT-based denoiser.

The repository contains code, configs, preprocessing utilities, and experiment launchers. Datasets, checkpoints, logs, generated samples, and Hydra output directories are intentionally excluded from Git.

## Main Features

- Heterogeneous graph diffusion with relation-family isolated edge spaces.
- pyHGT denoising with edge-phi fusion and dual softmax attention.
- Local block query training and sampling for large heterogeneous graphs.
- Subtype-aware node and edge modeling for ACM, DBLP, IMDB, PubMed, and HGHG-style subgraph datasets.
- Sampling and evaluation utilities for generated heterogeneous graphs.

## Repository Layout

```text

├── dihug/                  # Module entrypoint: python -m dihug.main
```

## Data

Data is not included in this repository. Put prepared datasets under `data/`.

Most heterogeneous subgraph datasets use this structure:

```text
data/<DatasetName>/
├── subgraph_000/
│   ├── nodes.pt
│   ├── edges.pt
│   └── meta.json
├── subgraph_001/
│   └── ...
└── processed/              # Generated by dataset loaders
```

The raw and generated directories below are ignored by Git:

```text
data/
output/
outputs/
logs/
run_logs/
runs/
```

## Installation

Create a Python environment. The repository includes a minimal conda template:

```bash
conda env create -f environment.yml
conda activate dihug
```

Install PyTorch and PyG for your own CUDA or CPU setup, following their official selectors. Then install the project:

```bash
pip install -r requirements.txt
pip install -e .
```

Some metrics use ORCA. Build it once if needed:

```bash
g++ -O2 -std=c++11 -o sparse_diffusion/analysis/orca/orca sparse_diffusion/analysis/orca/orca.cpp
```

The launcher scripts source `scripts/activate_env.sh`. They use the active conda environment when one is already active; otherwise they try `dihug`. Override the name or skip activation when needed:

```bash
CONDA_ENV=my_env bash scripts/run_dblp_single.sh
DIHUG_SKIP_CONDA=1 bash scripts/run_dblp_single.sh
```

Optional LLM-assisted preprocessing requires a Gemini key:

```bash
export GEMINI_API_KEY=...
```

## Quick Smoke Test

Run one epoch on a single graph:

```bash
python -m dihug.main +experiment=dblp_single_train train.n_epochs=1 train.batch_size=1 general.gpus=0
```

## Common Runs

```bash
bash scripts/run_acm_single.sh
bash scripts/run_dblp_single.sh
bash scripts/run_imdb_single.sh
bash scripts/run_pubmed_single.sh
```

Pass Hydra overrides after the script name:

```bash
CUDA_VISIBLE_DEVICES=0 bash scripts/run_pubmed_single.sh train.n_epochs=50 model.n_layers=4
```

Run PubMed baseline conversion and training:

```bash
bash scripts/run_pubmed_baseline_single.sh
```

Test from a checkpoint:

```bash
bash scripts/run_pubmed_test.sh /path/to/last.ckpt
```

## Preprocessing

Useful preprocessing and conversion scripts live in `dataprocess/scripts/`, for example:

```bash
python dataprocess/scripts/preprocess_pubmed.py
python dataprocess/scripts/preprocess_imdb.py
python dataprocess/scripts/preprocess_dblp_rule_based.py
python dataprocess/scripts/convert_hgen_pubmed_baseline.py
```

See the script headers and `dataprocess/scripts/README_*.md` files for dataset-specific inputs.

## Notes

This codebase inherits parts of SparseDiff, especially the discrete diffusion framework, metrics, and molecular/graph utility modules. DiHuG-specific work is centered on heterogeneous graph data handling, block-query training, and pyHGT denoising.

Relative dataset paths such as `data/ACM_subgraphs` resolve under the repository root by default. Processed caches are written next to the dataset, for example `data/ACM_subgraphs/processed/`. Older layouts with data placed beside the repository are still accepted when that path already exists.

Large local artifacts are ignored by `.gitignore`, including checkpoints (`*.ckpt`, `*.pt`), generated samples (`*.pkl`, `*.npy`, `*.jsonl`), processed data, and experiment outputs.
